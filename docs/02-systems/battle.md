# 战斗系统设计

## 概述

回合制卡牌战斗，玩家通过出牌和指挥角色行动来击败敌人。

## 战斗准备

- 玩家从已有角色中选择 **1-3 个**角色登场
- 正式战斗最多 **3 人上场**；序章教程例外：教程战 1 固定 1 人，教程战 2 固定 2 人
- 每个角色上场时为**满状态**（生命值等全满）
- 本次战斗的损失**不影响**下一次战斗

## 能量系统（已确定 v2）

- **公共能量池**，能量上限 **8 点**
- 打出卡牌消耗公共能量，**出牌不消耗 AS**
- **普通攻击不消耗能量**，每次普攻恢复 **1 点能量**
- 普攻回能**立即生效**
- 未用完的能量可累积到下一回合（不超过上限 8）

### 能量恢复（ER）

- 初始 ER = **0**（无自然恢复，必须靠普攻赚能量）
- ER 是可成长属性，通过肉鸽选项（稀有级）和局外成长提升
- **无硬上限**，每多 1 ER = 每回合多 1 点被动能量
- 核心博弈：**前期全靠普攻"打工赚能量"，ER 成长后逐步解放**

### AS 行动点系统（已确定 v2）

AS（Action Slot）**仅限制普攻次数**：

- **普攻**消耗 **1 AS**，回复 1 能量
- **出牌不消耗 AS**，仅消耗能量
- AS = 该角色每回合能普攻几次 = **经济产能**（每 AS 赚 1 能量）
- AS 用完后该角色不能再普攻，但仍可使用其卡牌

### AS 成长

- AS 成长 = **经济引擎升级**（每多 1 AS = 每回合多赚 1 能量）
- 通过肉鸽选项（史诗/传说级稀有度）获得
- **无硬上限**，用选项稀有度控制平衡

### 手牌系统

- 每回合抽 **3** 张牌
- 手牌上限 **8** 张（超过上限不抽）

### 能量策略

- **开局困境**：回合 1 能量=0，必须先普攻赚能量再出牌
- **蓄力爆发**：前几回合多普攻少出牌，攒满能量后一回合倾泻手牌
- **稳定循环**：每回合普攻赚 3 能量（3 名角色各 AS=1），打 2-3 张牌；AS/ER 成长后逐步提高出牌密度
- **AS 即产能**：所有角色初始 AS=1，多动通过卡牌效果（as_gain/as_reset）实现
- **出牌无 AS 限制**：手牌和能量够就能一直出，带来组合连锁的爽感

## 回合流程

```
抢攻阶段（仅第一回合）：
  全场 SPD 最高的单位获得 1 次额外行动
  - 我方：从该角色卡池随机复制 3 张牌，选 1 张免费使用或普攻 1 次（不消耗 AS 和能量）
  - 敌方：进行 1 次攻击或技能
  - SPD 并列时我方优先
  ↓
回合开始：恢复公共能量，触发回合开始效果
  ↓
抽牌阶段：从牌库中抽取手牌（有手牌数量上限）
  ↓
我方行动阶段 — 玩家自由执行（不限顺序）：
  - 打出手牌（仅消耗公共能量）
  - 指挥角色普通攻击（消耗 1 AS，回复 1 能量）
  - 每个角色普攻次数 ≤ 该角色当前 AS（出牌不受 AS 限制）
  ↓
敌方行动阶段 — 敌人执行 AI 行为
  ↓
回合结束：触发回合结束效果
  ↓
判定：检查胜负条件
```

## 角色属性（已确定）

| 属性 | 缩写 | 说明 |
|------|------|------|
| 生命值 | HP | 降为 0 时角色阵亡 |
| 攻击力 | ATK | 影响普通攻击和物理类伤害的基础值 |
| 法术强度 | AP | 影响卡牌伤害的基础值 |
| 防御力 | DEF | 减少受到的伤害 |
| 速度 | SPD | 决定抢攻权（仅第一回合，全场最高 SPD 获得 1 次额外行动） |
| 行动点 | AS | 每回合可普攻的次数（仅限制普攻次数） |
| 暴击率 | CRIT | 触发暴击的概率 |
| 暴击伤害 | CDMG | 暴击时的伤害倍率（如 150%） |
| 闪避 | EVD | 闪避敌方攻击的概率 |
| 命中 | ACC | 攻击命中敌方的概率 |

### 双攻体系说明

- **ATK（攻击力）**：影响普通攻击伤害，以及物理类卡牌的伤害
- **AP（法术强度）**：影响卡牌技能伤害（法术类卡牌）
- 不同角色侧重不同：有的角色 ATK 高适合普攻流，有的 AP 高适合出牌流
- 这为阵容搭配和卡牌构建提供了更多策略维度

## 牌库系统（已确定）

- 所有出战角色的卡牌**混合成公共牌库**
- 每个角色有**携带卡牌上限**，控制牌库规模
- 四区模型：牌库（Deck）→ 手牌（Hand）→ 弃牌堆（Discard）→ 消耗堆（Exhaust）
- 每回合从 Deck 抽 **3 张**牌到 Hand
- 手牌上限 **8 张**（超出时新抽的牌自动弃置）
- 使用后卡牌进入 Discard（消耗类卡牌进入 Exhaust）
- Deck 空时，Discard 洗牌回 Deck

## 速度与抢攻机制（已确定）

- 我方行动阶段内，玩家自由操控所有角色（不限顺序）
- **抢攻机制**：**仅第一回合**，全场 SPD 最高的单位获得 1 次额外行动
  - 我方获得抢攻：从该角色的卡池中随机复制 3 张牌，选 1 张免费使用或普攻 1 次
  - 敌方获得抢攻：敌方进行 1 次攻击或技能
  - SPD 并列时我方优先
  - 抢攻行动不消耗 AS 和能量
  - 抢攻抽取的卡牌独立于主牌库，用完即消失
  - 该角色卡牌不足 3 张时有几张抽几张
- 抢攻阶段结束后，固定为我方行动 → 敌方行动的顺序

## 胜负条件

- **胜利**：全部敌人被消灭
- **失败**：我方角色全部阵亡

## 伤害计算公式体系（已确定 v3）

### 设计原则

- **极致简洁**，玩家可直接心算实际伤害
- 所有角色初始 DEF = 0，防御通过卡牌/装备/肉鸽选项获取
- 普通攻击、卡牌伤害、卡牌治疗均可暴击

---

### 一、基础伤害公式

#### 1.1 普通攻击

```
伤害 = ATK - 目标DEF
```

直接减法，所见即所得。不消耗能量，是无牌可出时的兜底手段。

#### 1.2 卡牌伤害

```
伤害 = 卡牌伤害值 - 目标DEF
```

卡牌伤害值由卡牌自身描述决定（可能基于 ATK 或 AP 计算），最终结果减去目标 DEF。

#### 1.3 治疗

```
治疗量 = 卡牌治疗值
```

治疗值由卡牌自身描述决定（可能基于 AP 计算），治疗不受 DEF 影响。

---

### 二、防御减伤公式

采用**减法模型**，简洁直观：

```
最终伤害 = max(1, 伤害 - 目标DEF)
```

- 所有角色初始 DEF = 0，意味着早期伤害 = 攻击力，便于玩家直观计算
- DEF 通过卡牌效果、装备、肉鸽选项等途径获取
- 最低伤害保底 **1 点**，防止完全免伤
- **毒伤害无视 DEF**，不走防御减伤流程

#### 最低伤害保底

```
最终伤害 = max(1, 最终伤害)
```

任何伤害经过防御计算后，最低造成 **1 点伤害**。

### 三、数值取整规则

**所有涉及数值计算的最终结果均四舍五入取整，不出现小数：**

- 伤害值：`round(计算结果)`
- 治疗值：`round(计算结果)`
- 毒层数：`round(计算结果)`
- 护盾值：`round(计算结果)`
- Buff/Debuff 数值：`round(计算结果)`

> 游戏内任何时刻，HP、伤害、治疗、层数等数值均为整数。

---

### 三、命中 / 闪避判定

```
HitRate = clamp(ACC - EVD, 10, 100)
```

- 减法模型，简洁直观
- 下限 **10%**：任何攻击至少有 10% 命中率（不会完全打不中）
- 上限 **100%**：不会超过必中
- 判定时生成 1~100 随机数，≤ HitRate 则命中

| 攻击方 ACC | 防御方 EVD | 命中率 |
|-----------|-----------|--------|
| 95 | 5 | 90% |
| 95 | 15 | 80% |
| 95 | 30 | 65% |
| 100 | 5 | 95% |
| 110 | 25 | 85% |
| 80 | 30 | 50% |
| 70 | 80 | 10%（保底） |

> **数值指导**：普通角色 ACC 约 90~100，EVD 约 5~15。高闪避定位的角色 EVD 可达 25~35，需要通过高 ACC 或命中 Buff 来克制。

---

### 四、暴击判定与暴击伤害

#### 4.1 暴击判定

```
暴击触发：random(1, 100) ≤ CRIT
```

- CRIT 为百分比值（如 5 表示 5%）
- 暴击判定在命中成功后进行
- **普通攻击、卡牌伤害、卡牌治疗均可触发暴击**

#### 4.2 暴击伤害/治疗

```
暴击后数值 = 过防后伤害（或治疗量） × CDMG / 100
```

- CDMG 为百分比值（如 150 表示 150% = 1.5 倍）
- 所有角色初始 CDMG = 150，即暴击造成 1.5 倍
- 暴击放大的是**过防后的伤害**或**治疗量**
- 暴击率和暴击伤害通过卡牌/装备/肉鸽选项提升

---

### 五、完整伤害结算流程

```
┌─────────────────────────────────────┐
│          发起攻击/出牌              │
└─────────────┬───────────────────────┘
              ▼
┌─────────────────────────────────────┐
│  Step 1  命中判定                   │
│  HitRate = clamp(ACC - EVD, 10, 100)│
│  未命中 → 显示 "MISS"，流程结束     │
└─────────────┬───────────────────────┘
              ▼ （命中）
┌─────────────────────────────────────┐
│  Step 2  计算基础伤害               │
│  普通攻击：ATK                      │
│  卡牌伤害：由卡牌描述决定           │
└─────────────┬───────────────────────┘
              ▼
┌─────────────────────────────────────┐
│  Step 3  防御减伤（减法）            │
│  Damage = max(1, 基础伤害 - 目标DEF)│
│  毒伤害 → 跳过此步（无视 DEF）      │
└─────────────┬───────────────────────┘
              ▼
┌─────────────────────────────────────┐
│  Step 4  暴击判定                   │
│  命中暴击 → Damage = Damage×CDMG/100│
│  普攻/卡牌伤害/卡牌治疗均可暴击     │
└─────────────┬───────────────────────┘
              ▼
┌─────────────────────────────────────┐
│  Step 5  取整 + 最终结算             │
│  FinalDamage = round(Damage)        │
│  目标 HP -= FinalDamage             │
│  HP ≤ 0 → 目标阵亡                 │
│  所有数值四舍五入，不出现小数        │
└─────────────────────────────────────┘
```

```
→ 非暴击 18 伤害，对 110 HP 约需 7 次普攻击杀
→ 伤害计算直观：攻击力就是伤害，有防御就减掉
```

## 护盾机制（已确定）

- 护盾值可叠加，**上限 = 该角色最大 HP 的 200%**
- 受到伤害时**先扣护盾，再扣 HP**
- 护盾**不会在回合结束时消失**，持续到被伤害消耗完
- 护盾**不能被治疗恢复**，只能通过卡牌效果获得
- 固定伤害也会先扣护盾

## 效果类型系统（已确定 v3）

### 流血（Bleed）— 层数机制（回合开始前结算）

- 流血以**层数**叠加，施加时受 AP 缩放：`appliedStacks = round(baseStacks + AP × apScaling)`
- **目标回合开始前**结算：**伤害 = 当前层数**（固定伤害，无视 DEF），然后**层数 - 1**
- 流血层可多次叠加，新施加的层数直接加到现有层数上
- 层数降至 0 时流血消失

### 毒（Poison）— 层数机制（回合结束后结算）

- 毒以**层数**叠加，施加毒层时受 AP 缩放：`appliedStacks = round(baseStacks + AP × apScaling)`
- **目标回合结束后**结算：**伤害 = 当前层数**（固定伤害，无视 DEF），然后**层数 - 1**
- 毒层可多次叠加，新施加的层数直接加到现有层数上
- 层数降至 0 时毒消失

> **流血 vs 毒的区别**：流血在回合开始前结算（先流血再行动），毒在回合结束后结算（先行动再中毒）。两者是独立 debuff，可以同时存在于同一目标上。

### 虚弱（Weakness）— 层数机制

- 虚弱以**层数**叠加
- 效果：目标造成的最终伤害 = `伤害 - 虚弱层数`（最低 1）
- 每回合结束时**层数 - 1**，降至 0 时消失
- 施加虚弱时受 AP 缩放：`appliedStacks = round(base + AP × apScaling)`

### 脆弱（Vulnerable）— 层数机制

- 脆弱以**层数**叠加
- 效果：目标受到的最终伤害 = `伤害 + 脆弱层数`
- 每回合结束时**层数 - 1**，降至 0 时消失
- 施加脆弱时受 AP 缩放：`appliedStacks = round(base + AP × apScaling)`

> **虚弱 vs 脆弱**：虚弱减少自身造成的伤害（攻击方 debuff），脆弱增加自身受到的伤害（防御方 debuff）。两者可以同时存在。

### 致命一击引爆毒层（poison_detonate）

- 读取目标身上所有毒层，每层造成 **300% 伤害**（固定伤害，无视 DEF）
- 总引爆伤害 = `当前毒层数 × 3`
- 引爆后**不清空毒层**，目标原有毒层继续保留并按毒机制正常结算

### 流血引爆（bleed_detonate）

- 消耗目标身上所有流血层，造成 **流血层数²** 的固定伤害（无视 DEF）
- 引爆后流血层归零

### 效果类型速查表

| 效果类型 | 说明 |
|----------|------|
| damage | 造成伤害（物理/法术/固定），可基于 ATK/AP/护盾值计算 |
| heal | 治疗（base + AP × apScaling），可暴击 |
| shield | 施加护盾（base + AP × apScaling） |
| shield_amplify | 已有护盾按百分比增加 |
| poison | 施加毒层（层数机制，无视 DEF） |
| bleed | 施加流血层（层数机制，无视 DEF） |
| weakness | 施加虚弱层（减少目标造成的伤害） |
| poison_detonate | 引爆毒层（每层 ×300% 伤害，不清空毒层） |
| bleed_detonate | 引爆流血层（层数²伤害） |
| taunt | 嘲讽（强制敌方攻击自己） |
| lifesteal | 吸血（恢复造成伤害的百分比 HP） |

| as_reset | 重置指定角色的 AS 为满值 |
| as_max_increase | 提升 AS 上限（持续到本轮战斗结束） |
| buff | 施加增益（ATK/AP/DEF 等，可持续回合或整场战斗） |
| permanent_buff | 永久增益（持续到本轮战斗结束） |
| stat_multiply | 属性翻倍（如 ATK×2） |

| draw | 抽牌 |

| damage_reflect | 受攻击时反弹伤害 |
| card_transform | 将卡牌变为其他卡牌（持续到战斗结束） |
| percentage_hp_reduction | 减少目标当前 HP 百分比 |
