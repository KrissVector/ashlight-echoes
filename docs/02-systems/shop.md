# 商店系统

> 版本：v1.0
> 状态：当前有效
> 目的：统一商店节点的商品、服务、价格、刷新和机器内容表规则

## 概述

商店是 run 中的非战斗补给节点。它通过地图候选节点随机出现，不按固定间隔出现；第 1 章第 12 步至少出现 1 个商店选项，作为 Boss 前补给机会。

商店的核心作用是把金币转换为装备、药水、安全性和存档机会。商店不承担直接改卡、直接删卡或提供其它特殊商品线的职责。

## 维护规则

商店相关设定只在本文维护。其它文档需要提到商店商品、服务、价格、品质分布、禁止项或 `shops.json` 规则时，必须引用本文，不应重复维护同一组规则。

## 职责边界

商店系统负责：

- 商店出售哪些商品类型。
- 每章商店商品品质分布。
- 商品价格、回收价格和服务价格。
- 商店服务列表。
- 商店机器内容表 `shops.json` 的规则。

商店系统不负责：

- 商店节点何时出现；节点出现权重由 `map-encounters.md` 和 `run-flow.md` 定义。
- 装备本体效果；装备定义来自 `equipment.json`。
- 药水本体效果；药水定义来自 `potions.json`。
- 战斗奖励掉落；奖励表来自 `reward-tables.json`。
- 事件节点内容；事件节点应由独立事件内容表定义。

## 进入与离开流程

进入商店节点时：

1. 读取当前章节可用的 `ShopDef`。
2. 按商品规则生成本次商店库存。
3. 把生成结果写入当前节点状态。
4. 进入商店界面。

商店库存生成后必须随 run 存档保存。读档时不得重新刷新商店商品。

离开商店后进入节点完成流程。商店本身不发放战斗奖励。

## 出售商品

商店只出售两类商品：

| 商品类型 | 内容 | 数据来源 |
|----------|------|----------|
| 装备 | 通用装备，购买后按装备获得流程处理 | `equipment.json` 与装备池 |
| 药水 | 永久成长药水或一次性战斗药水，购买后进入全队药水背包 | `potions.json` 与药水池 |

商店明确不出售：

- 卡牌。
- 直接删牌服务。
- 装备和药水之外的其它商品类型。

卡牌复制、删除和改造只能通过药水、装备、事件或奖励等正式效果间接发生。商店可以出售具有卡牌操作效果的药水，但商店界面不能把这些效果包装成直接买卡、删卡或改卡服务。

## 装备商品

装备商品从对应品质的装备池抽取。

每个普通商店固定刷新 3 件装备。

| 品质 | 装备池 |
|------|--------|
| 普通 | `pool_equipment_common_all` |
| 稀有 | `pool_equipment_rare_all` |
| 史诗 | `pool_equipment_epic_all` |
| 传说 | `pool_equipment_legendary_all` |

商店不出售诅咒装备。诅咒装备只作为 Boss 奖励等特殊来源出现；商店只提供诅咒装备处理服务。

购买装备时：

1. 玩家选择商品和装备目标角色。
2. 若目标角色有空装备槽，装备进入空槽。
3. 若目标角色装备槽已满，玩家必须选择替换 1 件已装备装备，或取消购买。
4. 购买确认后扣除金币。
5. 装备后锁定，后续只能通过新装备替换或商店回收处理。

装备附带卡牌必须跟随装备来源。装备被替换或回收时，由该装备添加的卡牌同步移除。

## 药水商品

每个普通商店固定刷新 3 瓶药水。

药水商品从对应品质的药水池抽取。

| 品质 | 药水池 |
|------|--------|
| 普通 | `pool_potions_common_all` |
| 稀有 | `pool_potions_rare_all` |
| 史诗 | `pool_potions_epic_all` |
| 传说 | `pool_potions_legendary_all` |

购买药水时：

1. 若药水背包未满，购买确认后扣除金币并加入背包。
2. 若药水背包已满，玩家必须选择丢弃已有 1 瓶药水并购买，或取消购买。
3. 购买后的药水不能在商店界面直接使用；具体使用时机由 `PotionDef.useTiming` 和 [药水系统](potions.md) 决定。

## 商品品质分布

### 装备

商店装备品质按章节分布：

| 章节 | 装备品质分布 |
|------|--------------|
| 第1章 | 普通 100% |
| 第2章 | 普通 80% / 稀有 20% |
| 第3章 | 普通 60% / 稀有 30% / 史诗 10% |
| 第4章 | 普通 40% / 稀有 40% / 史诗 20% |
| 第5章 | 普通 30% / 稀有 40% / 史诗 25% / 传说 5% |

第 5 章商店传说装备最多出现 1 次。出现过后，本 run 后续第 5 章商店不再生成传说装备。

### 药水

第 1 章 MVP 商店药水品质为普通 100%。

第 2-5 章商店药水品质分布尚未定死。补后续章节商店规则时，必须在本文件和 `shops.json` 中明确写出，不能直接套用装备品质表。

## 价格

装备和药水使用同一套品质价格区间。

| 品质 | 购买价格 | 回收价 |
|------|----------|--------|
| 普通 | 25-50 | 售价的 40% |
| 稀有 | 50-80 | 售价的 40% |
| 史诗 | 100-180 | 售价的 40% |
| 传说 | 200-350 | 售价的 40% |
| 诅咒 | 不出售 | 花费 25 金币处理 |

若装备或药水内容表中已经写明 `price`，以内容表价格为准。

`price.min` 表示购买价格下限，`price.max` 表示购买价格上限，`price.sellRatio` 表示回收比例。

## 商店服务

商店提供以下服务：

| 服务 | 价格 | 说明 |
|------|------|------|
| 手动存档 | 10 金币 | 写入手动存档槽，不覆盖自动存档 |
| 装备回收 | 售价的 40% | 回收已装备装备，装备槽变为空槽 |
| 药水回收 | 售价的 40% | 回收背包中的药水 |
| 诅咒装备处理 | 25 金币 | 移除诅咒装备，不获得金币 |
| 永久 debuff 净化 | 25 金币 | 清除允许商店净化的永久型节点修正 |
| 装备修理 | 25 金币 | 修复被节点修正失效的装备 |

如果某个永久型 debuff 在自身定义中写了不同的 `cleanseRule`，以该修正的机器内容表为准。

## 第 1 章 MVP 商店规则

第 1 章商店必须支持：

- 固定刷新 3 件装备，使用普通装备池，品质普通 100%。
- 固定刷新 3 瓶药水，使用普通药水池，品质普通 100%。
- 服务常驻：手动存档、装备回收、药水回收、诅咒装备处理、永久 debuff 净化、装备修理。

## 机器内容表

商店机器内容表应位于：

```text
docs/03-content/data/shops.json
```

`shops.json` 至少需要表达 `ShopDef`。

```ts
interface ShopDef {
  id: ShopId;
  chapterId: ChapterId;
  itemRules: ShopItemRuleDef[];
  services: ShopServiceDef[];
}

interface ShopServiceDef {
  type: ShopServiceType;
  price: ShopServicePrice;
}

type ShopServicePrice =
  | number
  | { type: "sell_ratio"; sellRatio: number };
```

字段说明：

| 字段 | 中文说明 | 用途 |
|------|----------|------|
| `id` | 商店定义唯一标识 | 节点绑定具体商店规则 |
| `chapterId` | 适用章节 | 限制商店在哪个章节出现 |
| `itemRules` | 商品生成规则列表 | 生成装备和药水商品 |
| `services` | 商店服务列表 | 生成存档、回收、净化、修理等服务 |
| `itemType` | 商品类型 | 只能是 `equipment` 或 `potion` |
| `minCount` / `maxCount` | 最少 / 最多生成数量 | 控制每类商品刷新数量 |
| `rarityWeights` | 品质权重 | 按章节决定普通、稀有、史诗、传说概率 |
| `poolId` | 内容池标识 | 指向装备池或药水池 |
| `price` | 服务价格 | 固定价格或价格规则 |
| `price.type` | 价格规则类型 | `sell_ratio` 表示按商品售价比例回收 |
| `sellRatio` | 回收比例 | 例如 `0.4` 表示售价的 40% |

商店数据校验必须检查：

- `itemType` 只能是 `equipment` 或 `potion`。
- `poolId` 必须存在，且池内条目类型和 `itemType` 一致。
- 商店不得包含卡牌商品、直接删牌服务或装备和药水之外的其它商品类型。
- 商品生成结果必须写入存档，读档不得重新随机。
- 第 1 章商店商品池非空。
- 第 1 章普通商店固定生成 3 件装备和 3 瓶药水。
