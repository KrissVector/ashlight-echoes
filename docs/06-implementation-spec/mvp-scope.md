# MVP 范围

## 目标

第一版实现必须支持从全新开局到第 1 章结束的完整可玩流程。

目标玩家流程：

```text
新游戏
  -> 序章教程战 1：玲纱单人
  -> 序章剧情：灼夜加入
  -> 序章教程战 2：玲纱 + 灼夜
  -> 第 1 章「银誓余烬 / 银誓要塞遗址」随机候选节点路线
  -> 第 1 章步 6 固定灼夜追杀剧情节点（血战到底！ / 突出重围）
  -> 第 1 章 Boss：断誓之盾·莱恩哈特
  -> 第 1 章通关结算、存档、第 2 章占位解锁
```

只有当以上流程可以在不使用 debug 工具的情况下完整打通时，MVP 才算完成。

## 必须实现的章节

### 序章

必须实现：

- 线性流程，不提供路线选择。
- 教程战 1：只上场玲纱。
- 剧情过渡：灼夜加入队伍。
- 教程战 2：玲纱和灼夜共同上场。
- 教程提示可以简化，但玩家必须学会：
  - 普攻消耗 AS，并立即获得能量
  - 卡牌消耗公共能量
  - 多名角色共享一个牌库和一个能量池
  - 结束回合后进入敌方行动

### 第 1 章

必须实现：

- 第 1 章总计 13 步。
- 章节标题为「银誓余烬」，主地点为「银誓要塞遗址」。
- 第 13 步固定为第 1 章 Boss。
- 首次游玩第 1 章时，第 6 步固定为灼夜追杀剧情节点，固定提供「血战到底！」和「突出重围」两个分支。
- 选择「血战到底！」时，固定应用「血战到底」战场修正；先按正常规则抽取起手手牌，再解锁玲纱大招 `immovable` 与灼夜大招 `flame_raven`，并把两张新解锁的大招各加入手牌和本场战斗牌库。
- 选择「血战到底！」并胜利后，要记录剧情标记 `shakuya_amulet_origin_revealed`，表示她得知父母遗物护身符很可能来自故乡地下古代遗迹，是背后势力想夺回研究的完成品样本；同时给灼夜 1 次大招升级机会。
- 选择「突出重围」时，固定应用「突出重围」剧情修正：我方开场生命值为最大生命值的 80%；该分支不立即解锁大招，玲纱和灼夜的大招在第 1 章 Boss 战开场解锁。
- 路线选择采用 `map-encounters.md` 中的随机候选节点模式：每步按权重随机生成 2/3 个候选节点，玩家从候选中选择 1 个进入。
- 第 1 章通关后保存进度，并清晰结束 MVP 流程。

第一版不要求支持“通关后重复游玩第 1 章”的完整规则；如果存档系统自然支持，可以保留。

## 必须实现的可玩角色

MVP 必须可玩：

- 玲纱
- 灼夜

MVP 中锁定、不可玩：

- 初叶
- 瑠璃
- 影月

这些角色的数据可以保留在正式内容表中，但 MVP UI 不应允许玩家选择她们上场。

## 必须实现的卡牌

必须实现：

- 玲纱和灼夜的全部固有初始卡牌。
- 大招锁定和解锁机制。
- 玲纱大招和灼夜大招在第 1 章步 6「血战到底！」分支战斗开始时解锁；若玩家选择「突出重围」，则两人大招在第 1 章 Boss 战开场解锁。解锁战斗均先抽起手手牌，再把新解锁的大招加入手牌和牌库，不占用起手抽牌数量。
- 「血战到底！」胜利后，触发 1 次灼夜大招升级选择；「突出重围」不发放该升级机会。
- 共享牌库、手牌、弃牌堆、消耗堆。
- 每回合抽 3 张，手牌上限 8 张。
- 卡牌费用、目标校验、效果执行。

卡牌升级系统必须实现，因为正式卡牌内容表已经包含升级定义。第 1 章是否发放升级奖励由奖励表或 run 流程定义；没有正式奖励入口时，不应临时随机发放升级。

## 必须实现的敌人

必须实现的敌人内容：

- 序章教程敌人：蹒跚活尸、灰壳侵染体、烬齿幼狼。
- 第 1 章普通敌人：蹒跚活尸、灰壳侵染体、烬齿幼狼、食腐尸鬼、染疫尸鬼、荒原尸鹫、灰烬骑士、灰誓弩手、空甲守卫、虚缝烬犬、癫狂的强盗（仅「血战到底！」胜利后进入普通战池）。
- 第 1 章精英敌人：银誓断刃、缚誓壁垒、钟楼缚魂师。
- 第 1 章特殊精英：癫狂的强盗首领、癫狂的强盗。该战叙事目标是血旗佣兵团追夺灼夜身上的父母遗物护身符。
- 第 1 章 Boss：断誓之盾·莱恩哈特。

必须实现的敌人变体：

- 序章教程战 1：`shambling_corpse` ×1。
- 序章教程战 2：`ash_husk` ×1 + `cinderfang_whelp` ×2。
- 第 1 章普通战：从普通战遭遇池随机绑定。
- 第 1 章精英战：从精英战遭遇池随机绑定。
- 第 1 章第 6 步「血战到底！」：`maddened_bandit_leader` ×1 + `maddened_bandit` ×4。
- 第 1 章第 6 步「突出重围」：`maddened_bandit` ×3，我方开场 HP 为最大 HP 的 80%。
- 「血战到底！」胜利后，`maddened_bandit` 可作为普通敌人出现在本 run 后续第 1 章普通战中。
- 第 1 章 Boss 战：`oathbreaker_reinhardt` 三阶段 AI。
- 莱恩哈特二阶段召回 `ashen_knight`。

敌人 AI 必须数据驱动，或至少隔离在清晰模块中，确保后续章节能新增敌人而不重写战斗主循环。敌人技能触发必须使用“条件筛选 + 权重随机”：先筛出当前可用技能，再按 `weight` 抽取，不使用固定优先级顺序。

癫狂强盗系敌人的“虚空侵蚀”必须实现：攻击命中时，若目标在命中前没有护盾，则施加 2 层脆弱和 2 层虚弱；自身每次被攻击命中时，获得 1 层脆弱和 1 层虚弱。

## 必须实现的战斗系统

必须实现：

- 公共能量池，默认能量上限为 8。
- ER 初始为 0。
- 普攻消耗 1 AS，并立即获得 1 点能量。
- 出牌不消耗 AS。
- 每回合刷新 AS。
- 第一回合抢攻机制。
- 命中 / 闪避判定。
- 暴击判定和暴击伤害。
- 伤害和治疗最终结果取整数。
- DEF 使用减法模型，伤害最低为 1。
- 护盾机制，包括护盾上限为最大 HP 的 200%。
- 流血、毒、虚弱、脆弱。
- 攻击附加流血时，必须在伤害结算后目标没有护盾才会成功施加。
- 敌方默认单体目标选择使用角色存在感权重；基础数值读取 `docs/03-content/data/characters.json` 的 `baseStats.presence`。
- `presence_gain` 存在感增益效果；具体卡牌效果读取 `docs/03-content/data/cards.json`。
- `mark` 标记效果；癫狂的强盗首领会通过「猎杀标记」施加标记并优先攻击标记目标。
- `poison_detonate` 读取毒层造成伤害，但不清空毒层。
- `bleed_detonate` 消耗流血层。
- 若 MVP 卡牌或奖励会用到，则必须支持：嘲讽、吸血、抽牌、能量获得、AS 获得 / 重置、buff、卡牌变形、消耗 / Exhaust。
- 全部敌人被击败时胜利。
- 我方全部角色阵亡时失败。
- 每场战斗开始时，上场角色 HP 回满；随后再应用战斗开始修正，例如「突出重围」会把我方当前 HP 调整为最大 HP 的 80%。

第一版战斗动画可以简化，但状态变化必须可见、可理解。

## 必须实现的 run 系统

必须实现：

- 新 run 创建。
- 序章完成后进入第 1 章。
- 第 1 章前段二选一、后段三选一的随机候选节点路线选择。
- 固定步骤覆盖：
  - 首次第 1 章第 6 步：灼夜追杀剧情节点，固定提供「血战到底！」和「突出重围」
  - 第 13 步：Boss
- 节点类型：
  - 普通战斗
  - 精英战斗
  - 事件
  - 商店
  - Boss
- `node-modifiers.md` 中可在第 1 章出现的节点修正。
- 战后随机事件只有在 `run-flow.md` 或事件表正式定义触发规则后才进入 MVP；未定义前不做临时随机触发。
- 节点之间可以使用药水。
- 进入第 1 章节点前按规则选择队伍，最多 3 人。MVP 通常只有玲纱 + 灼夜。

## 必须实现的奖励

必须实现：

- 战后属性成长三选一。
- 若经济规则启用，则支持单个选项刷新。
- 金币奖励。
- 药水掉落奖励。
- 精英和 Boss 装备掉落。
- 第 1 章 Boss 奖励流程使用当前 Boss 奖励规则。
- 第 1 章通关结果界面。

奖励系统必须支持第 1 章会用到的全部稀有度，包括 Boss 奖励需要的稀有度。

## 必须实现的装备

必须实现的机制：

- 每名角色 3 个通用装备槽。
- 获得装备时立即决定是否装备。
- 装备后锁定。
- 装备槽满时提供替换流程。
- 装备属性加成。
- 如果 MVP 装备池使用装备附带卡牌或改牌效果，则必须支持对应机制。
- 如果第 1 章 Boss 奖励可能提供诅咒装备，则必须支持诅咒装备处理。
- 如果商店内容启用装备出售 / 回收，则必须支持对应流程。

必须实现的内容：

- 第 1 章商店装备池。
- 第 1 章精英掉落装备池。
- 灼夜特殊精英奖励装备池。
- 第 1 章 Boss 奖励装备池。

具体第 1 章装备池应在编写物品数据前补齐。

## 必须实现的药水

必须实现的机制：

- 全队共享药水背包，容量上限为 3。
- 药水只能在节点之间使用。
- 药水效果永久、不可撤销。
- 需要目标时，使用药水时选择目标角色。
- 背包满时，获得新药水需要选择丢弃旧药水或放弃新药水。
- 如果第 1 章 Boss 奖励或商店配置可能出现传说药水，则必须支持传说药水效果。

必须实现的内容：

- 第 1 章药水掉落池。
- 第 1 章商店药水池。
- Boss 药水奖励池。

商店不直接卖卡，也不提供直接删牌服务。复制、删除、改造卡牌通过药水效果间接实现，尤其是传说药水。

## 必须实现的商店

必须实现：

- 商店节点 UI。
- 购买药水。
- 购买装备。
- 如果当前规则启用，则支持装备 / 药水回收。
- 如果第 1 章可能出现永久 debuff，则支持永久 debuff 净化 / 修理。
- 花费金币手动存档。
- 如果第 1 章定义了具体情报效果，则支持情报购买。

明确不允许：

- 直接购买卡牌。
- 直接删牌服务。

## 必须实现的事件

必须实现：

- 事件节点框架。
- 事件选项和结果。
- 事件结果可以影响金币、药水、节点修正或其他 run 状态。

内容依赖：

- 第 1 章需要补齐具体事件列表，才能算内容完整。

## 必须实现的存档 / 进度

必须实现：

- 新 run 存档状态。
- 序章完成或进入第 1 章后的自动存档。
- 第 1 章 Boss 胜利后的自动存档。
- 商店手动存档。
- 继续已保存 run。
- run 失败结果。
- 第 1 章通关结果。

MVP 的局外成长可以非常轻量，只需要保存足够状态，让系统知道第 1 章已通关，并能在未来解锁第 2 章内容。

## 必须实现的 UI 界面

必须实现：

- 标题 / 开始 / 继续。
- 序章剧情对话。
- 战斗界面。
- 手牌和目标选择。
- 结束回合按钮。
- 战斗结果。
- 奖励选择。
- 药水背包和药水使用。
- 装备获得和替换。
- 路线选择。
- 事件节点。
- 商店。
- 存档 / 读档反馈。
- 第 1 章通关结果。
- run 失败结果。

缺少最终美术资源时可以使用占位资源，但布局应遵循 `docs/04-art/style-guide.md`。

## 明确不在第一版范围内

第一版不要求实现：

- 第 2-5 章可玩内容。
- 初叶、瑠璃、影月作为可玩角色。
- 完整长期局外成长树。
- 无尽模式。
- 每日挑战或种子分享系统。
- 最终生产级动画质量。
- 完整本地化管线。
- 移动端专项优化。
- 运营或商业化系统。

## MVP 验收标准

MVP 不单独维护 `acceptance-cases.md`。第一版上线或形成可玩版本后，以项目负责人实际游玩体验为主要验收方式；以下条目作为游玩验收时必须覆盖的标准。

满足以下条件时，MVP 视为通过：

- 玩家可以从新游戏开始，不借助开发工具进入第 1 章。
- 序章能教会普攻回能和出牌。
- 玩家可以在第 1 章从随机生成的候选节点中进行路线选择。
- 普通战、精英战、商店、事件、固定剧情战、Boss 节点都能正常完成。
- 战斗失败会结束 run，并进入失败结果。
- Boss 胜利会结束第 1 章，并进入通关结果和保存进度。
- 玲纱和灼夜的大招解锁时机符合设计。
- 奖励、装备、药水、金币会影响后续第 1 章战斗。
- 存档和继续游戏可以恢复第 1 章进行中的 run。
- 不需要第 2 章内容也不会卡死流程；显示“未开放 / coming next”占位状态即可。
